üéÆ Game Mechanics

Today we'll learn to make a real game with scoring, collectibles, and rules! We'll create a game where you collect items to earn points.

üöÄ Activity 1: Create a Score

Step 1: Open MakeCode Arcade and create a new project called "Scoring Game"
Step 2: Create a player sprite that you can move around
Step 3: Look in the "Info" category
Step 4: Find "set score to 0" and add it in "on start"
Step 5: The score will automatically appear on screen!
The score starts at 0 and you can change it. It will display in the top corner of the screen automatically!

üé® Activity 2: Create Collectible Items

Step 1: Create a new sprite for a collectible item (like a coin or star)
Step 2: Set its kind to "Food" (in Sprites category)
Step 3: Position it somewhere on the screen
Step 4: Add an animation to make it look interesting (like spinning or pulsing)
Step 5: Create multiple collectibles at different positions!

üéØ Activity 3: Collect Items and Score

Step 1: Look for "on sprite of kind Player overlaps otherSprite of kind Food" (in Sprites category)
Step 2: This block runs when your player touches a collectible!
Step 3: Inside it, add "change score by 1" (from Info category)
Step 4: Add "destroy otherSprite" to make the item disappear when collected
Step 5: Add a sound effect (like "coin" or "power up")
Step 6: Test it - when you touch a collectible, your score increases!

üåü Activity 4: Spawn New Items

Step 1: After an item is collected, create a new one
Step 2: Use "set newSprite to sprite of kind Food"
Step 3: Use "set newSprite position to x (pick random 0 to 160) y (pick random 0 to 120)"
Step 4: Now new items appear randomly when you collect one!

üéÆ Activity 5: Win Condition

Step 1: Add a win condition - when score reaches 10
Step 2: In your game loop, check if score equals 10
Step 3: If true:
  • Show text "You Win!"
  • Play victory music
  • Stop the game
Step 4: Use "game.over(true)" to end the game with a win

‚ö†Ô∏è Activity 6: Obstacles and Losing

Step 1: Create enemy sprites (kind: Enemy)
Step 2: Make them move automatically (in game loop)
Step 3: Add collision detection: "on sprite of kind Player overlaps otherSprite of kind Enemy"
Step 4: When player touches enemy:
  • Show "Game Over!"
  • Play sad sound
  • End game with "game.over(false)"
Create a complete game:
  • Player moves around collecting items
  • Score increases when collecting items
  • New items spawn randomly
  • Enemies move around the screen
  • Game ends if you touch an enemy
  • Win when you collect 20 items
  • Show high score
  • Different items worth different points!

üí° Key Concepts

  • Score: A number that tracks player progress
  • Collectible: Items the player can pick up
  • Collision Detection: Detecting when sprites touch each other
  • Sprite Kinds: Categories for sprites (Player, Enemy, Food, etc.)
  • Game Over: Ending the game (win or lose)
  • Spawning: Creating new sprites during the game
  • Random: Using random numbers for unpredictable gameplay
