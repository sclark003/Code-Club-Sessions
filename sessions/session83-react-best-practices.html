Step 1: Keep components small and focused:

  return <button onClick={onClick}>{text}</button>;
}

// Bad: Component does too much

  // 200 lines of code doing everything
}

ğŸ“‹ Complete Example Program

Example: Here's a well-organised React app structure: 
// components/Button/Button.js
import './Button.css';

function Button({ children, onClick, variant = 'primary' }) {
  return (
    <button 
      className={`btn btn-${variant}`}
      onClick={onClick}
    >
      {children}
    </button>
  );
}

export default Button;

// components/Card/Card.js
import './Card.css';

function Card({ title, children }) {
  return (
    <div className="card">
      {title && <h3 className="card-title">{title}</h3>}
      <div className="card-content">
        {children}
      </div>
    </div>
  );
}

export default Card;

// App.js
import Button from './components/Button/Button';
import Card from './components/Card/Card';

function App() {
  return (
    <div className="App">
      <Card title="Welcome">
        <p>This is a well-organised React app!</p>
        <Button onClick={() => alert('Clicked!')}>
          Click Me
        </Button>
      </Card>
    </div>
  );
}
Refactor a component to:
  • Break it into smaller components
  • Extract reusable parts
  • Organise files properly
  • Add proper prop types or comments

ğŸ’¡ Key Concepts

  • Component organisation: Keep related files together
  • Small components: Easier to understand and reuse
  • Naming: Use clear, descriptive names
  • Props: Pass data down, not up
  • Reusability: Make components flexible with props
  • Comments: Explain why, not what
