üéÆ Remote Control Robot

Today we'll turn your micro:bit into a remote control for Cutebot! You'll press buttons to make the robot move, turn, and stop.

üöÄ Activity 1: Button A = Forward

Step 1: Open MakeCode and create a new project called "Cutebot Buttons"
Step 2: Make sure the Cutebot extension is added (see Session 13 if needed)
Step 3: Add an "on button A pressed" block (from Input category)
Step 4: Inside it, add "move forward at speed 50%" (from Cutebot category)
Step 5: Download and test - press button A and Cutebot should move forward!
The robot will keep moving until you tell it to stop! We'll add a stop command next.

üé® Activity 2: Complete Remote Control

Step 1: Add "on button B pressed" to make Cutebot move backward
Step 2: Add "on button A+B pressed" to make Cutebot stop
Step 3: Now you have:
  • Button A = Forward
  • Button B = Backward
  • Buttons A+B = Stop
Step 4: Test your remote control!

üéØ Activity 3: Turn Controls

Step 1: We need to use tilt controls for turning. Add "on tilt left" (from Input)
Step 2: Inside it, add "turn left at speed 50%"
Step 3: Add "on tilt right" with "turn right at speed 50%"
Step 4: Add "on tilt forward" to stop (or use it for forward movement)
Step 5: Now you can:
  • Tilt micro:bit left = Turn left
  • Tilt micro:bit right = Turn right
  • Button A+B = Stop

üåü Activity 4: Display Status

Step 1: Add "show string" blocks to display what's happening
Step 2: When button A is pressed, show "F" (Forward)
Step 3: When button B is pressed, show "B" (Backward)
Step 4: When A+B is pressed, show "S" (Stop)
Step 5: Now the micro:bit screen shows what command you're sending!
Create an advanced remote control:
  • Button A = Forward at speed 30
  • Button B = Backward at speed 30
  • Shake = Emergency stop (stop immediately)
  • Tilt left = Turn left
  • Tilt right = Turn right
  • Show different icons for each action!

üí° Key Concepts

  • Remote Control: Controlling something from a distance using buttons or sensors
  • Event-Driven: Actions happen when events occur (like button presses)
  • Continuous Movement: The robot keeps moving until told to stop
  • User Interface: Using the micro:bit display to show status information
  • Input to Output: Button press (input) ‚Üí Robot movement (output)
