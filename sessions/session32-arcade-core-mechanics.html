Today we'll build the core mechanics of your game - the foundation that everything else will build on. We'll create a solid, well-organized code structure.

üöÄ Activity 1: Project Setup

Step 1: Open MakeCode Arcade and create a new project
Step 2: Name it after your game (from your design document)
Step 3: Set up the basic structure in "on start":
  • Set background
  • Create player sprite
  • Set initial game state variables
  • Initialize score and lives
Step 4: Create variables for game state:
  • "gameState" (playing, paused, gameOver, menu)
  • "score"
  • "lives"
  • "level"

üé® Activity 2: Player Setup

Step 1: Create your player sprite with the design from your document
Step 2: Set its position (start position)
Step 3: Set its kind to "Player"
Step 4: Add physics properties:
  • Set acceleration (how fast it speeds up)
  • Set max speed (how fast it can go)
  • Set bounce (if it should bounce off walls)
Step 5: Add animation (walking, idle, etc.)

üéØ Activity 3: Advanced Movement System

Step 1: Create a function called "updatePlayerMovement"
Step 2: Inside, use controller inputs to set velocity:
  • If left pressed: set vx (velocity x) to negative
  • If right pressed: set vx to positive
  • If up pressed: set vy (velocity y) to negative
  • If down pressed: set vy to positive
  • If nothing pressed: gradually slow down
Step 3: Call this function in "game.onUpdate"
Step 4: Add boundary checks to keep player on screen
Step 5: Test - movement should feel smooth and responsive!

üåü Activity 4: Game State Management

Step 1: Create functions for different game states:
  • "startGame" - Initialize everything
  • "updateGame" - Main game loop
  • "pauseGame" - Pause everything
  • "gameOver" - Handle game over
Step 2: In "game.onUpdate", check gameState:
  • If "playing": run updateGame
  • If "paused": don't update
  • If "gameOver": show game over screen
Step 3: Add button to pause/unpause game

üéÆ Activity 5: Collision System

Step 1: Set up collision handlers for different sprite kinds
Step 2: Create collision for Player + Collectible:
  • Increase score
  • Play sound
  • Destroy collectible
  • Spawn new collectible
Step 3: Create collision for Player + Enemy:
  • Decrease lives
  • Play hurt sound
  • Brief invincibility (can't be hurt for a moment)
  • If lives = 0, game over
Step 4: Add visual feedback for collisions (flash, particles)

‚ö° Activity 6: Lives System

Step 1: Create a variable "lives" and set to 3 in "on start"
Step 2: Display lives on screen (use text sprite or info.setLife)
Step 3: When player hits enemy:
  • Decrease lives by 1
  • Update display
  • If lives = 0, call gameOver function
Step 4: Add invincibility period after being hit (so player doesn't lose all lives instantly)
Build a complete core game system with:
  • Player with smooth movement and animation
  • Game state management (playing, paused, game over)
  • Lives system with visual display
  • Scoring system
  • Collision detection for collectibles and enemies
  • Boundary checking
  • Pause functionality

üí° Key Concepts

  • Game Loop: Code that runs continuously to update the game
  • Game State: What mode the game is in (playing, paused, etc.)
  • Velocity: Speed and direction of movement (vx, vy)
  • Physics: How objects move and interact (acceleration, friction, etc.)
  • Collision Detection: Detecting when sprites touch
  • Invincibility: Temporary protection from damage
  • Function Organization: Breaking code into reusable functions
