ğŸš€ Activity 1: Tuples

Step 1: Tuples are like lists, but they can't be changed (immutable): 
# Create a tuple
coordinates = (10, 20)
print(coordinates)

# Tuple with multiple items
person = ("Sarah", 12, "London")
print(person)

# Access items (like lists)
print(person[0])  # Sarah
print(person[1])  # 12

# Tuples can't be modified
# person[1] = 13  # This would cause an error!
Step 2: Tuples are useful when you want data that shouldn't change: 
# RGB colour values
red = (255, 0, 0)
green = (0, 255, 0)
blue = (0, 0, 255)

# Coordinates
point1 = (3, 4)
point2 = (7, 2)
Tuples use parentheses () instead of square brackets []. They're faster than lists and use less memory!

ğŸ’» Activity 2: Tuple Operations

Step 1: You can do some operations with tuples: 
tuple1 = (1, 2, 3)
tuple2 = (4, 5, 6)

# Concatenate
combined = tuple1 + tuple2
print(combined)  # (1, 2, 3, 4, 5, 6)

# Repeat
repeated = tuple1 * 2
print(repeated)  # (1, 2, 3, 1, 2, 3)

# Check if item exists
print(2 in tuple1)  # True

# Count and index
numbers = (1, 2, 3, 2, 4, 2)
print(numbers.count(2))  # 3
print(numbers.index(3))  # 2

ğŸ¨ Activity 3: Sets

Step 1: Sets store unique items (no duplicates) and are unordered: 
# Create a set
fruits = {"apple", "banana", "orange"}
print(fruits)

# Add items
fruits.add("grape")
print(fruits)

# Remove items
fruits.remove("banana")
print(fruits)

# Sets automatically remove duplicates
numbers = {1, 2, 2, 3, 3, 3, 4}
print(numbers)  # {1, 2, 3, 4}
Step 2: Sets are great for finding unique items: 
# Find unique items in a list
numbers = [1, 2, 2, 3, 3, 3, 4, 4]
unique = set(numbers)
print(unique)  # {1, 2, 3, 4}

# Convert back to list
unique_list = list(unique)
print(unique_list)

ğŸ“Š Activity 4: Set Operations

Step 1: Sets support mathematical operations: 
set1 = {1, 2, 3, 4, 5}
set2 = {4, 5, 6, 7, 8}

# Union (all items from both)
union = set1 | set2
print(union)  # {1, 2, 3, 4, 5, 6, 7, 8}

# Intersection (items in both)
intersection = set1 & set2
print(intersection)  # {4, 5}

# Difference (items only in first set)
difference = set1 - set2
print(difference)  # {1, 2, 3}

# Check if subset
print({1, 2} <= set1)  # True

ğŸ¯ Activity 5: When to Use Each

Step 1: Choose the right data structure: 
# Use a LIST when:
# - You need to modify items
# - Order matters
# - You need duplicates
shopping_list = ["milk", "bread", "eggs"]

# Use a TUPLE when:
# - Data shouldn't change
# - Order matters
# - You want it to be faster
coordinates = (10, 20)

# Use a SET when:
# - You need unique items
# - Order doesn't matter
# - You need set operations
unique_tags = {"python", "coding", "fun"}

Step 1: Create a program that finds common interests: 
# Students and their interests
student1_interests = {"coding", "reading", "music", "sports"}
student2_interests = {"coding", "music", "art", "reading"}
student3_interests = {"sports", "music", "coding", "gaming"}

common_1_2 = student1_interests & student2_interests
print(f"Student 1 and 2 both like: {common_1_2}")

# Find all unique interests
all_interests = student1_interests | student2_interests | student3_interests
print(f"All interests: {all_interests}")

# Find interests only student 1 has
only_student1 = student1_interests - student2_interests - student3_interests
print(f"Only student 1 likes: {only_student1}")

ğŸ’¡ Key Concepts

  • Tuple: An immutable (unchangeable) ordered collection
  • Set: An unordered collection of unique items
  • Immutable: Cannot be changed after creation
  • Unique: No duplicate items allowed
  • Union (|): Combines items from both sets
  • Intersection (&): Items that are in both sets
  • Difference (-): Items only in the first set
  • When to use: Lists for modifiable data, tuples for fixed data, sets for unique items
