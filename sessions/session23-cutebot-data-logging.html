üìä Data Logging

Today we'll learn to record and analyze data from your robot! This helps you understand what your robot is doing and improve its performance.

üöÄ Activity 1: Basic Data Logging

Step 1: Create lists to store data:
  • "distanceReadings" - Store ultrasonic readings
  • "lineReadings" - Store line sensor values
  • "timeStamps" - Store when each reading was taken
Step 2: In your main loop, record data:
  • Read sensor value
  • Add to list
  • Record time
Step 3: Limit list size (only keep last 50 readings)
Step 4: Display data on micro:bit screen when button pressed

üé® Activity 2: Distance Logging

Step 1: Log distance readings while robot moves
Step 2: Record:
  • Distance value
  • Time of reading
  • Robot position (if tracking)
Step 3: Analyze the data:
  • Find minimum distance (closest obstacle)
  • Find maximum distance (furthest point)
  • Calculate average distance
Step 4: Display statistics on screen

üéØ Activity 3: Line Following Analysis

Step 1: Log line sensor readings while following
Step 2: Track:
  • Left sensor values
  • Right sensor values
  • When line was lost
  • When line was found
Step 3: Analyze:
  • How often line is lost
  • Which sensor detects line more
  • Patterns in line detection

üåü Activity 4: Performance Metrics

Step 1: Track performance metrics:
  • Time to complete task
  • Number of obstacles encountered
  • Number of times line was lost
  • Total distance traveled
Step 2: Calculate efficiency:
  • Speed (distance / time)
  • Success rate (tasks completed / attempts)
  • Error rate (mistakes / total actions)
Step 3: Display performance summary

üéÆ Activity 5: Data Visualization

Step 1: Create simple graphs on micro:bit screen
Step 2: Display distance over time as a bar graph
Step 3: Show trends (increasing, decreasing, stable)
Step 4: Use LEDs to visualize data patterns

üìà Activity 6: Using Data to Improve

Step 1: Analyze logged data to find problems
Step 2: Identify patterns:
  • Where does robot get stuck?
  • What causes errors?
  • What works well?
Step 3: Adjust code based on findings
Step 4: Test again and compare data
Create a comprehensive data logging system:
  • Logs data from multiple sensors
  • Stores data in organized lists
  • Calculates statistics (min, max, average)
  • Displays data visually on screen
  • Tracks performance metrics
  • Uses data to improve robot behavior

üí° Key Concepts

  • Data Logging: Recording information during operation
  • List/Array: Storing multiple values
  • Statistics: Analyzing data (min, max, average)
  • Performance Metrics: Measurements of how well robot performs
  • Data Analysis: Looking for patterns and insights
  • Visualization: Displaying data in visual form
  • Iterative Improvement: Using data to make robot better
