üí° What is a Light Sensor?

The micro:bit uses its LED display as a light sensor! When the LEDs aren't lit up, they can detect how much light is shining on them. The light level is measured from 0 (very dark) to 255 (very bright).

üöÄ Activity 1: Display Light Level

Step 1: Open MakeCode and create a new project called "Light"
Step 2: Go to the "Input" category and find "light level"
Step 3: Add an "on button A pressed" block
Step 4: Add "show number" and drag "light level" into it
Step 5: Press button A and see the light level number!
Step 6: Try covering the micro:bit with your hand - the number should go down. Shine a light on it - the number should go up!
Light levels range from 0 (pitch black) to 255 (very bright). Normal room light is usually around 50-150.

üé® Activity 2: Light Meter

Step 1: Add a "forever" loop
Step 2: Inside, add "plot bar graph" from the Led category
Step 3: Drag "light level" into the first slot, and set the maximum to 255
Step 4: Add a pause. Now you have a visual light meter - the more bars, the brighter it is!

üéØ Activity 3: Dark Detector

Step 1: Create a new project called "DarkDetector"
Step 2: Add a "forever" loop
Step 3: Inside, add an "if" block
Step 4: Check if "light level" is less than (<) 50 (dark)
Step 5: If dark, show an icon (like a moon or star)
Step 6: Add "else" and show a different icon for light (like a sun)
Step 7: Add a pause. Cover the micro:bit to see it change!

üåü Activity 4: Night Light

Step 1: Create a project that turns on LEDs when it gets dark
Step 2: Use an if statement to check if light level < 30
Step 3: If dark, use "plot bar graph" with a high value to light up the screen
Step 4: If light, clear the screen
Create a smart light that:
  • Shows a dim light (low brightness) when it's slightly dark
  • Shows a bright light when it's very dark
  • Shows nothing when it's bright
  • Hint: Use multiple if/else statements!

üí° Key Concepts

  • Light Level: A number from 0-255 representing how bright it is
  • Sensor Reading: Getting a value from a sensor (like light level)
  • Conditional Logic: Making decisions based on sensor values
  • Comparison Operators: Less than (<), greater than (>), equal to (=)
