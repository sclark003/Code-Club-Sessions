üéÆ Game Controls

Today we'll learn to make your sprites move when you press buttons! The Arcade console has directional buttons (up, down, left, right) that we can use to control our game characters.

üöÄ Activity 1: Move with Buttons

Step 1: Open MakeCode Arcade and create a new project called "Movement"
Step 2: Create a sprite in "on start" (like we did last session)
Step 3: Look for the "Controller" category in the blocks menu
Step 4: Find "on left button pressed" block
Step 5: Drag it onto your workspace
Step 6: Inside it, add "change mySprite x by -5" (from Sprites category)
Step 7: Test it - press the left button and your sprite should move left!
Negative numbers move left (or up), positive numbers move right (or down). The number is how many pixels to move!

üé® Activity 2: All Directions

Step 1: Add "on right button pressed" with "change mySprite x by 5"
Step 2: Add "on up button pressed" with "change mySprite y by -5"
Step 3: Add "on down button pressed" with "change mySprite y by 5"
Step 4: Now you can move in all four directions!
Step 5: Try different numbers (like 10 or 2) to make movement faster or slower

üéØ Activity 3: Smooth Movement

Step 1: Find the "game.onUpdate" block (in Game category)
Step 2: This block runs continuously - perfect for smooth movement!
Step 3: Inside it, add an "if" statement
Step 4: Check "controller left is pressed" (from Controller category)
Step 5: If true, move sprite left by -2
Step 6: Add similar checks for right, up, and down
Step 7: Now movement is smooth and continuous while you hold the button!

üåü Activity 4: Keep Sprite on Screen

Step 1: Add boundary checks to keep your sprite on screen
Step 2: After moving, check if sprite x position is less than 0 (off left edge)
Step 3: If so, set sprite x to 0 (keep it on screen)
Step 4: Check if sprite x is greater than 160 (off right edge) - set to 160
Step 5: Do the same for y (top: 0, bottom: 120)
Step 6: Now your sprite can't leave the screen!
Create a game where:
  • Player sprite moves smoothly in all directions
  • Sprite stays on screen (can't go off edges)
  • Different speeds for different directions (faster left/right, slower up/down)
  • Button A makes sprite jump (move up quickly then down)
  • Button B makes sprite dash (move faster temporarily)

üí° Key Concepts

  • Button Events: Code that runs when a button is pressed
  • Game Loop: Code that runs continuously (like game.onUpdate)
  • Coordinates: X (left/right) and Y (up/down) positions
  • Boundaries: Limits that keep sprites on the screen
  • Pixel Movement: Moving by a certain number of pixels
  • Continuous vs. Single: Holding button (continuous) vs. pressing once (single)
